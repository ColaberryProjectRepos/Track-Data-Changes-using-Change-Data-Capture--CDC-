What is Change Data Capture (CDC) ? 1

What is  Change Data Capture (CDC)?

CDC Stands for Change Data Capture. It tracks changes being made to the tables or databases.  Anytime when there is update, delete, and Inserted information into the tables or databases, it would capture those changes and mirror them into separate tables. It manages the log sequence number and use them to restore and  works independently of triggers.  It can be enabled only using system stored procedures.

How CDC works ?

1- Inserts, updates, and deletes are applied to tracked source tables. 

2- The changes are loaded to the transaction log.

3- Capture process  reads the logs and sends change information to change tables. 

4- Function identify changes over specified ranges, it would show as a filtered result set.

5- The result set can be used by another process.
