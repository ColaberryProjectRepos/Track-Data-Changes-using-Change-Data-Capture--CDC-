How to enable Change Data Capture (CDC ) as a table level using SQL Server? 3

After a database has been enabled for change data capture, the db_owner fixed database role can create a capture instance for individual source tables by using the stored procedure sys.sp_cdc_enable_table. 

Run the script from the uploaded Data File to create the Global_Store table before you enable CDC for the table.

Additionally, it is very important to understand the role of the required parameter @role_name. If there is any restriction of how data should be extracted from database, this option is used to specify any role which is following restrictions and getting access to data to this option if there is one.  If you do not specify any role and, instead, pass a NULL value, data access to this changed table will not be tracked and will be available to access by everybody.

Run the script from the uploaded Data File to create the Global_Store table. Then, use the below query  will enable at a table level.

  1. USE CDC
  2. GO
  4. EXEC  sys.sp_CDC_enable_table 
  5. @Source_schema = N'dbo',
  6. @Source_name = N'Glob_Store',
  7. @Role_name = Null



The table which have been created are listed here.

  • cdc.captured_columns – This table returns result for list of captured column.
  • cdc.change_tables – This table returns list of all the tables which are enabled for capture.
  • cdc.ddl_history – This table contains history of all the DDL changes since capture data enabled.
  • cdc.index_columns – This table contains indexes associated with change table.
  • cdc.lsn_time_mapping – This table maps LSN number (for which we will learn later) and time.

When everything is successfully completed,  check  the system tables again and you will find a new table  called cdc.dbo_Glob_store_CT. 

The below link will demonstrate how to enable CDC at database level and table level.
